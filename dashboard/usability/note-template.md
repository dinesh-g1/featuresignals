## Severity Scale

| Rating | Definition | Example |
|---|---|---|
| **1 — Cosmetic** | Minor annoyance. Does not affect task completion. | Misaligned icon, typo |
| **2 — Moderate** | Causes hesitation or extra steps but task is completed. | Unclear label, extra click required |
| **3 — Serious** | Causes significant delay, errors, or requires workaround. | Missing confirmation, hidden feature |
| **4 — Critical** | Prevents task completion entirely. | Broken flow, crash, completely misleading UI |

**Action threshold:**
- Severity 1–2: Fix in next design iteration.
- Severity 3: Fix before release.
- Severity 4: Fix immediately, block release.
